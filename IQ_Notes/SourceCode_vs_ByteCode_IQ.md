# Source Code vs Byte Code — IQ Note

## The One-Liner

**Source code** is what *humans write* (readable, high-level). **Byte code** is what *machines interpret* (optimized, low-level). JavaScript engines (V8, SpiderMonkey) compile source → byte code before execution.

---

## Breakdown Table

| Layer | Source Code | Byte Code |
|-------|------------|-----------|
| **Who reads it** | Humans | JS Engine (V8) |
| **Format** | Text (`.js` files) | Binary-like instructions |
| **Readability** | ✅ Full English, syntax | ❌ Opaque numeric opcodes |
| **Editability** | ✅ You write & edit it | ❌ Generated, not hand-edited |
| **Size** | Larger (whitespace, comments) | Smaller (compressed, no fluff) |
| **Speed** | Slow to parse/execute directly | Fast to execute (pre-compiled) |
| **Where it lives** | On disk / in your editor | In memory (after parsing) |
| **Example** | `console.log("Hello")` | `0x2a 0x01 0x0d "Hello"` |

---

## Example Walkthrough — `helloworld.js`

### Step 1: You write source code

```js
// helloworld.js
console.log("Hello, World!");
```

This is **source code** — plain text you can open in Notepad, edit, read, and understand immediately.

### Step 2: Node.js (via V8) parses & compiles it

When you run:

```bash
node helloworld.js
```

Here's what happens under the hood:

```
┌─────────────────────────────────────────────────────────────┐
│                     node helloworld.js                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
              ┌──────────────────────────────┐
              │  1. PARSING                  │
              │  Source → AST (Abstract      │
              │  Syntax Tree)                │
              │  Checks syntax validity      │
              └──────────┬───────────────────┘
                         │
                         ▼
              ┌──────────────────────────────┐
              │  2. COMPILATION              │
              │  AST → Byte Code             │
              │  (V8's Ignition interpreter) │
              └──────────┬───────────────────┘
                         │
                         ▼
              ┌──────────────────────────────┐
              │  3. EXECUTION                │
              │  Byte Code → CPU runs it     │
              │  (Hot paths → TurboFan       │
              │   optimizes to Machine Code) │
              └──────────────────────────────┘
```

### Step 3: The engine sees byte code

You never *see* the byte code directly — V8 keeps it in memory. But if you dump it, it looks something like this:

```
[generated bytecode for: console.log("Hello, World!")]
Parameter count 0
Frame size 8
         0x2a LdaConstant [0]          // Load "Hello, World!"
         0x2c CallRuntime [console.log] // Call the function
         0x2f Return                    // Return
```

## Pipeline Diagram

```
┌────────────┐     ┌──────────┐     ┌────────────┐     ┌─────────────┐
│ SOURCE     │────▶│ PARSER   │────▶│ COMPILER   │────▶│ INTERPRETER │
│ helloworld │     │ (Scanner │     │ (Ignition  │     │ + JIT       │
│ .js        │     │  + Parser│     │  → Byte    │     │ (TurboFan   │
│ (text)     │     │  → AST)  │     │  Code)     │     │  → Machine  │
└────────────┘     └──────────┘     └────────────┘     │  Code)      │
                                                        └─────────────┘
```

| Stage | Input | Output | What happens |
|-------|-------|--------|--------------|
| **Scanner** | Source text | Tokens | Breaks `console.log("Hi")` into tokens: `console`, `.`, `log`, `(`, `"Hi"`, `)` |
| **Parser** | Tokens | AST | Builds a tree: `CallExpression(console.log, ["Hi"])` |
| **Compiler** | AST | Byte Code | V8's Ignition generates compact byte code instructions |
| **Interpreter** | Byte Code | Execution result | Runs the byte code line-by-line |
| **JIT (TurboFan)** | Hot byte code | Machine Code | Optimizes frequently-run paths into raw CPU instructions |

---

## Key Takeaways

- **Source code** is the *what* (your intent). **Byte code** is the *how* (engine's plan).
- Engines don't run your `.js` directly — they compile it to byte code first, then run that.
- Byte code is **portable** — same `.js` file → different byte code per engine (V8 vs SpiderMonkey).
- The **JIT compiler** (TurboFan in V8) watches for "hot" (frequently run) byte code and compiles it all the way down to **machine code** for raw CPU speed.
- You can see V8 byte code yourself: `node --print-bytecode helloworld.js`

---

## TL;DR

> **Source code** = human-friendly text you write. **Byte code** = engine-friendly instructions the JS engine generates from it. `node helloworld.js` → parse → compile to byte code → execute. Byte code is faster to run but impossible for humans to write productively.
